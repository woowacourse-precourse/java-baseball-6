
# type : subject	-> 제목

# body (optional) 	-> 본문

# footer (optional)	-> 꼬리말

# -----------------------------
# type rule
# Feat:         새로운 기능 추가
# Fix:          버그 수정
# Docs :        문서 수정
# Style :       코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우
# Refactor :    코드 리펙토링
# Test :        테스트 (테스트 코드 추가, 수정, 삭제, 비즈니스 로직에 변경 없는 경우)
# Chore :       그 외 기타 변경사항 (빌드 스크립트 수정, 패키지 매니저 수정 등)
# Design :      CSS 등 사용자 UI 디자인 변경
# Comment :     필요한 주석 추가 및 변경
# Init :        프로젝트 초기 생성
# Rename :      파일 혹은 폴더명 수정하거나 옮기는 경우
# Remove :      파일을 삭제하는 작업만 수행하는 경우

# -----------------------------
# subject rule
# 제목은 최대 50 글자 넘지 않기
# 마침표 및 특수 기호 사용 X
# 첫글자 대문자, 명령문 사용, 동사 원형 사용, 과거 시제 사용 X
# 개조식 구문으로 작성

# -----------------------------
# body rule
# 한 줄당 72자 내로 작성
# 최대한 자세히 작성
# 어떻게 보다는 무엇을, 왜 변경 했는지에 대해 작성

# -----------------------------
# footer rule
# 유형: #이슈번호 형식으로 작성
# 이슈 트래커 ID 작성
# 여러 개의 이슈 번호는 ,로 구분
# 예시: Fixes: #45 Related to: #34, #23

# 이슈 트래커 유형
# Fixes:        이슈 수정중(아직 해결되지 않은 경우)
# Resolves :	이슈를 해결한 경우
# Ref :	        참조할 이슈가 있을 때 사용
# Related to :	해당 커밋에 관련된 이슈 번호(아직 해결되지 않은 경우)
