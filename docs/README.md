

## 기능 요구사항
1. 기본적으로 1부터 9까지 "서로 다른 수"로 이루어진 3자리의 수를 맞추는 게임이다.
   -> 즉 0이 들어가지 않음 + 중복 불가.
2. 조건
    * 스트라이크 : 같은 수 같은 자리
    * 볼 : 같은 수 다른 자리
    * 낫싱 : 다른 수 다른 자리
3. 사용자가 답을 맞출 때까지 입력하는 동작 방식.
   -> 답을 맞추면 사용자는 종료(1), 재도전(2)할 수 있음
   -> 옳지 않은 답 입력시 즉시 IllegalArgumentException을 발생		
   (예를 들어 3자리의 정수가 아닌 수, 문자형 등)

    
---

## 입출력 요구사항
* 입력
    * 서로 다른 3자리의 수
    * 게임이 끝난 경우 재시작/종료를 구분하는 1과 2 중 하나의 수

* 출력
  : 입력한 수에 대한 결과를 볼, 스트라이크 개수로 표시
  ex1) 1볼 1스트라이크
  ex2) 맞는 수가 하나도 없는 경우
  -> 낫싱
  ex3) 3개의 숫자를 모두 맞춘 경우
  -> 3스트라이크
  ->3개의 숫자를 모두 맞히셨습니다! 게임 종료
  ex4) 시작문구 출력
  ->숫자 야구 게임을 시작합니다.

코드를 List<Integer> computer = new ArrayList<>();
while (computer.size() < 3) {

## Application 클래스

### 개요

Application 클래스는 모든 객체의 생성과 의존성 주입을 담당하며, 게임의 실행 흐름을 시작합니다.


### 주요 기능

1. 객체의 생성과 초기화: Application 클래스는 다양한 클래스의 인스턴스를 생성하고 초기화합니다. 이것은 게임 애플리케이션의 필수 구성 요소를 준비하는 단계로, 예를 들어 Constant, CheckForm, IO, Game, GameConfiguration, 및 Runner 클래스의 인스턴스를 생성합니다.

2. 의존성 주입: Application 클래스는 생성된 객체 간의 의존성을 설정합니다. 이것은 한 객체가 다른 객체와 상호 작용하고 필요한 데이터나 기능을 공유할 수 있도록 하는 과정입니다. 예를 들어, Game 객체가 IO 객체를 사용하여 입출력 작업을 처리하기 위해 IO 객체를 주입받습니다.

3. 게임 실행 흐름 시작: Application 클래스는 Runner 객체를 생성하고 게임 실행을 시작하는 역할을 합니다. 이것은 게임 애플리케이션의 진입점이며, Runner 객체를 통해 게임 흐름을 관리하고 실행합니다.

4. 사용자 입력 및 결과 출력: Application 클래스는 직접 사용자 입력을 처리하거나 결과를 출력하지 않습니다. 이러한 역할은 IO 및 Game 객체에서 처리됩니다. IO 객체는 사용자로부터 입력을 받고, Game 객체는 게임 로직을 처리하며 결과를 출력합니다.

요약하면, Application 클래스는 애플리케이션의 초기화 및 실행 시작을 담당하며, 객체 간의 의존성을 설정하여 게임 애플리케이션의 흐름을 개시합니다.

## CheckForm 클래스
### 개요

CheckForm 클래스는 사용자의 입력에 대한 유효성을 검사하는 역할을 하는 클래스입니다. 이 클래스는 주로 게임의 규칙을 준수하도록 사용자의 입력을 검증하고, 그렇지 않다면 오류를 발생시켜 사용자에게 오류메시지를 전달합니다.


### 주요 역할
CheckForm 클래스는 다음과 같은 주요 역할을 수행합니다:

1. 사용자 입력의 유효성 검사: 사용자로부터 받은 입력값이 숫자인지, 길이가 3자리이며 중복된 숫자가 없는지 검사하여 게임 규칙을 준수하도록 합니다.

2. 게임 재시작 또는 종료 확인: 사용자가 정답을 맞춘 후, 다음 단계로 게임을 계속할지 종료할지를 결정하기 위해 1 또는 2의 숫자를 입력해야 함을 확인합니다.

## Constant 클래스

### 개요
Constant 클래스는 상수 값을 정의하고 이러한 상수 값을 제공하는 데 사용되는 클래스입니다. 이 클래스는 게임 상태를 나타내는 사용자 입력 관련 상수 값을 관리하는 데 사용됩니다.

### 주요 역할
Constant 클래스는 다음과 같은 주요 역할을 수행합니다:

1. 상수 관리: GAME_RESTART 및 GAME_END 상수를 정의하고 관리하여 게임 애플리케이션에서 사용자 입력에 대한 상수 값을 제공합니다.

2. 다른 클래스에서 사용: 다른 클래스에서 이 상수 값을 사용하여 사용자 입력을 확인하고 게임 상태를 변경하거나 게임을 다시 시작하거나 종료할 수 있도록 합니다.

Constant 클래스는 게임 애플리케이션에서 중요한 역할을 하며, 사용자 입력과 게임 상태를 관리하는 데 필수적인 상수 값을 제공합니다.

## Game 클래스
### 개요
Game 클래스는 게임 진행에 필요한 로직이 포함된 클래스로, 주로 숫자 야구 게임의 규칙을 구현하고 게임의 진행을 제어하는 데 사용됩니다


### 주요 역할

1. 게임 진행 로직 구현: judgeGame 메서드를 통해 사용자 입력과 정답을 비교하여 스트라이크와 볼을 계산하고 게임 결과를 판정합니다.

2. 정답 생성: createAnswer 메서드를 통해 게임의 정답을 생성하고 저장합니다.

3. 입출력 관리: io 속성을 통해 입출력을 처리하며, 결과를 출력하거나 사용자로부터 입력을 받는 데 사용됩니다.

Game 클래스는 숫자 야구 게임의 핵심 로직을 구현하고, 게임의 진행을 관리하는 역할을 합니다.

## GameConfiguration 클래스
### 개요
GameConfiguration 클래스는 게임 관련 객체 간의 의존성 주입 및 관리를 수행합니다. 즉 외부에서 필요로 하는 해당 객체들을 제공하고 의존성을 주입합니다. 또한 Application 클래스에서 생성자 주입을 효율적으로 수행하기 위한 역할을 합니다.


### 주요 역할

1. 의존성 주입 및 관리: 다른 객체인 Constant, CheckForm, Game에 대한 의존성을 관리하고, 각 객체를 생성하여 설정합니다.

2. 객체 간의 상호작용 관리: 객체들 간의 상호작용을 중개하고 게임의 진행을 조절하는 역할을 합니다.

3. 생성자 주입 단순화: GameConfiguration 클래스는 Application 클래스에서 생성자 주입을 단순화시킵니다. 필요한 객체들을 하나의 클래스로 묶어주어 Application 클래스에서 인수의 개수를 줄이는 데 도움을 줍니다.

이렇게, GameConfiguration 클래스는 객체 간의 의존성을 관리하고 게임의 진행을 원활하게 조절하는 중요한 역할을 합니다.

## IO 클래스
### 개요
IO 클래스는 게임의 입출력을 담당하는 역할을 합니다. 사용자로부터 숫자를 입력 받고, 게임의 결과를 출력하는 기능을 제공합니다.

### 주요 기능
1. 사용자 입력 받기
   String input(): 사용자로부터 숫자를 입력받아 반환합니다. 입력 프롬프트를 표시한 후, 사용자로부터 숫자 입력을 대기합니다.
2. 게임 결과 출력
   String success(): 게임에서 3개의 숫자를 모두 맞춘 경우, 해당 결과를 출력하고 게임 종료 여부를 사용자에게 묻습니다. 사용자의 입력을 위한 객체를 반환합니다.
3. 볼과 스트라이크 출력
   void BallStrikePrint(int strike, int ball): 볼과 스트라이크의 개수를 받아, 해당 결과를 출력합니다. 볼과 스트라이크가 모두 0보다 큰 경우, 볼과 스트라이크 수를 함께 출력합니다. 하나만 0보다 큰 경우, 해당 결과만을 출력하며, 모두 0인 경우 "낫싱"을 출력합니다.

IO 클래스는 게임 진행 중 사용자와의 상호작용을 처리하며, 게임 결과를 보기 좋게 출력하는 역할을 합니다.

## Runner 클래스
### 개요
Runner 클래스는 게임의 전체 진행을 관리하고 게임의 흐름을 조절합니다. 이 클래스는 게임을 시작하고, 게임 진행 중 사용자의 입력을 받아 게임 결과를 확인하며, 게임을 재시작하는 역할을 합니다.

### 주요 기능

1. 게임 시작 메시지 출력: 게임이 시작될 때 "숫자 야구 게임을 시작합니다." 메시지를 화면에 출력합니다.
2. 게임 진행 상태 판단: 게임 진행 중인지, 게임 종료인지 판단하기 위한 변수 "judge"를 활용합니다.
3. 랜덤한 정답 생성: 게임 시작 시, game 객체를 이용하여 랜덤으로 정답을 생성합니다.
4. 사용자 입력 처리 및 유효성 검사: IO 객체를 사용해 사용자로부터 입력을 받아 숫자 유효성을 검사합니다.
5. 게임 진행 : game 객체를 이용하여 사용자의 입력 숫자와 정답을 비교하여 스트라이크와 볼을 계산하고 결과를 출력합니다.
5. 게임 결과 확인 및 게임 종료 또는 재시작 처리: 게임 결과가 "SUCCESS"인 경우, 게임을 종료하거나 재시작하는 작업을 수행합니다.

### 주요 객체
1. game: 게임 객체
2. checkForm: 사용자 입력의 유효성 검사를 담당하는 객체
3. io: 사용자와의 입출력을 관리하는 객체

Runner 클래스는 게임 진행을 단순하게 조절하며, 게임의 시작, 진행, 결과 확인, 재시작을 관리합니다.
