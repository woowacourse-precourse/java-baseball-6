1. 컴퓨터가 가지는 렌덤의 수
    1.1. 같은 수를 가지면 안된다.
    1.2. 3자리를 가졍야 된다.

2. 플레이어가 입력하는 수
    2.1. 3자리 수를 입력 받을 수 있게 만든다.
    2.2. 잘못된 값(String, x<3 or x>3)을 받는 경우 예외

3. 컴퓨터와 플레이어의 수를 비교
    3.1. 같은 수에 같은 인덱스를 가지면 strike, 같은 수에 다른 인덱스를 가지면 ball, 나머진 낫싱
    3.2. 3스트라이크가 아닌경우 2번으로 롤백 

4. 재시작 여부
    4.1. 게임이 끝난 후 플레이어로 부터 입력값을 받고 1이면 재시작 2이면 종료
